Artifact Specification · packaging

SCORM as a
media container

Import the tested package as a page, not an assessment. The page editor can then adjust the embedded activity’s underlying width while the activity stays in its instructional context.

01

Minimal folder pattern

Zip the files inside the activity folder—not the containing folder.

activity-folder/
  index.html
  imsmanifest.xml
  assets/

The zip should open directly to index.html and imsmanifest.xml. Keep every local image, stylesheet, script, font, and media file in the package.

02

Minimal manifest shape

Ask the builder to create the complete SCORM 1.2 manifest and list every local asset.

<manifest ...>
  <resource ... href="index.html">
    <file href="index.html" />
    <file href="assets/name.ext" />
  </resource>
</manifest>
03

Test, package, import as a page, preserve

  1. Test editable source: keyboard, reset, screen size, and completion.
  2. Set index.html as the entry file.
  3. Create the manifest and list every local asset.
  4. Zip the package contents.
  5. Import as a page, not as an assessment.
  6. Open the page in the Rich Content Editor and adjust the embedded activity’s underlying width.
  7. Preview as a student, then add instructions and follow-up where the learning sequence needs them.
  8. Preserve source, assets, design record, prompts, and tested package.

Guardrails

Keep the SCORM activity ungraded. Use a Canvas quiz, assignment, discussion, or rubric when assessment is needed. Keep the deployable zip separate from editable project files, and retain both outside Canvas.
